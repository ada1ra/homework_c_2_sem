# О работе ДКА

## Алфавит 

В структуре `Transition` как `symbol` используются именно эти символы.

Символы алфавита:
- 'd' - цифра
- '.' - точка
- 'E' - экспонента
- '+' - плюс
- '-' - минус

## Состояния:

- 0: начальное
- 1: после минуса
- 2: целая часть
- 3: точка без целой части
- 4: точка после целой части
- 5: дробная часть
- 6: после E
- 7: после знака экспоненты
- 8: экспонента

### Допускающие: 
- 2 (целое число)
- 5 (дробное число)
- 8 (число с экспонентой)

### Таблица переходов:

| Состояние | d | . | E | + | - |
|-----------|---|---|---|---|---|
| 0         | 2 | 3 | - | - | 1 |
| 1         | 2 | - | - | - | - |
| 2         | 2 | 4 | 6 | - | - |
| 3         | 5 | - | - | - | - |
| 4         | 5 | - | - | - | - |
| 5         | 5 | - | 6 | - | - |
| 6         | 8 | - | - | 7 | 7 |
| 7         | 8 | - | - | - | - |
| 8         | 8 | - | - | - | - |

Прочерк `-` означает отсутствие перехода (переход в состояние ошибки)

## Код ошибки

В функции `dfaAccept` значение `errorCode` обозначает:
- `0` - успех
- `1` - ошибка (недопустимый символ, например, `,`)
- `2` - ошибка (невозможен переход, например, `.` в состоянии экспонента)
